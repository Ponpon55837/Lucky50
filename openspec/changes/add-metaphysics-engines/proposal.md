# 新增傳統命理引擎系統

## Why

Lucky50 目前的運勢計算僅使用基礎的五行能量和生肖加成，系統中已有但未啟用的命理數據（納音五行、五行相生相剋、生肖配對、星座）以及更進階的命理系統（八字十神、紫微斗數、風水方位增強、流日吉凶）都未整合。這些系統能顯著提升投資建議的深度和個人化程度，讓運勢分析從「泛用型」升級為「個人命理定制型」。

## What Changes

**變更 ID**: `add-metaphysics-engines`
**目標系統**: Lucky50 - 農民曆智慧投資系統
**變更類型**: 新增功能 + 架構擴展
**優先級**: 高

- **新增**: `MetaphysicsEngine` 插件式架構介面
- **新增**: 八字十神引擎 — 投資性格分析
- **新增**: 紫微斗數引擎（簡化版）— 命宮/財帛宮/事業宮主星分析
- **新增**: 風水方位引擎增強 — 流日吉凶 + 個人方位
- **新增**: 納財吉日引擎 — 財運日曆
- **修改**: 啟用現有未使用的納音五行、五行相生相剋、生肖配對、星座
- **修改**: `IntegratedFortuneService` 改為透過引擎聚合器收集各引擎結果
- **修改**: `UserProfile` 增加 `birthPlace`（選填，風水方位用）
- **修改**: Dashboard UI 增加各命理系統展示區塊

## Impact

- **影響規格**: fortune, architecture, user
- **影響程式碼**: `src/services/integratedFortune.ts`, `src/services/fortune.ts`, `src/types/index.ts`, `src/views/Dashboard.vue`
- **影響範圍**: 新增約 5-7 個 service 檔案，修改 3-4 個現有檔案，新增約 200+ 單元測試

## 問題分析

### 現有系統的未使用資產

| 資產         | 位置                       | 狀態                   |
| ------------ | -------------------------- | ---------------------- |
| 納音五行     | `lunar.getYearNaYin()`     | 有取得，未影響計算     |
| 五行相生相剋 | `ELEMENT_RELATIONS`        | 已定義，eslint-disable |
| 生肖配對     | `ZODIAC_COMPATIBILITY`     | 已定義，eslint-disable |
| 星座         | `lunar.getConstellation()` | 有取得，未整合         |

### 缺失的命理系統

1. **八字十神** — 日主與其他干支的十神關係，直接映射投資性格
2. **紫微斗數** — 14主星分布，分析財富/事業傾向
3. **風水方位增強** — 結合月柱、個人元素的每日方位
4. **流日吉凶** — 日干支與個人八字的交互分析
5. **納財吉日** — 特定干支組合的財運日

## 解決方案

### 核心架構：MetaphysicsEngine 插件系統

```
┌─────────────────────────────────────────────┐
│          MetaphysicsEngineRegistry          │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐       │
│  │ BaZi    │ │ ZiWei   │ │ FengShui│ ...   │
│  │ TenGods │ │ Engine  │ │ Engine  │       │
│  │ Engine  │ │         │ │         │       │
│  └────┬────┘ └────┬────┘ └────┬────┘       │
│       │           │           │             │
│       ▼           ▼           ▼             │
│  ┌─────────────────────────────────────┐    │
│  │    MetaphysicsResult (標準化輸出)    │    │
│  └─────────────────────────────────────┘    │
│                    │                        │
│                    ▼                        │
│  ┌─────────────────────────────────────┐    │
│  │   IntegratedFortuneService (聚合)    │    │
│  └─────────────────────────────────────┘    │
└─────────────────────────────────────────────┘
```

### 五大命理引擎

#### 引擎 1：已啟用的現有系統（Phase 1）

- 納音五行 → 影響五行能量權重
- 五行相生相剋 → 調整分數
- 生肖配對 → 社交 compatibility
- 星座 → 補充分析

#### 引擎 2：八字十神（Phase 2）

- 計算日主（出生日天干）
- 分析四柱干支與日主的十神關係
- 映射到投資性格：
  - 偏財旺 → 積極型交易者
  - 正財旺 → 穩健型投資者
  - 食神旺 → 研究型分析者
  - 傷官旺 → 逆向思考型
  - 比肩/劫財旺 → 風險偏好型
  - 正官/七殺旺 → 紀律型
  - 正印/偏印旺 → 學習型

#### 引擎 3：紫微斗數簡化版（Phase 3）

- 使用已有出生日期+時間（不需額外輸入）
- 計算流程：
  1. 農曆生日 → 定命宮位置
  2. 命宮干支 → 查14主星分布
  3. 財帛宮/事業宮 → 投資傾向分析
- 簡化：不排完整命盤，只取主要星曜
- 輸出：命宮主星、財帛宮主星、事業宮主星

#### 引擎 4：風水方位增強 + 流日吉凶（Phase 4）

- 方位：結合月柱天干 + 個人元素 + 當日地支
- 流日：日干支與個人八字的相合/相沖分析
- 納財吉日：特定干支組合標記

#### 引擎 5：插件架構 + 使用者設定（Phase 5）

- `MetaphysicsEngine` 介面標準化
- 使用者可啟用/停用個別引擎
- 可調整各引擎權重
- Dashboard UI 整合展示

### 使用者資料需求

**不需額外輸入**：所有命理計算都從現有的 `UserProfile`（姓名、出生日期、出生時間）推算。

**選填欄位**：

- `birthPlace`（出生地城市名）— 僅風水方位增強用，不影響核心計算

## 成功指標

1. 所有現有 198+ 測試繼續通過
2. 每個新引擎有獨立單元測試（目標：每個引擎 15-25 個測試）
3. 總測試數達到 300+
4. 使用者可在設定中啟用/停用各引擎
5. Dashboard 展示所有啟用引擎的結果
6. 所有計算從現有 UserProfile 推算，不需額外輸入
7. README.md 內容與專案實際狀態完全一致

### 整合要求（每 Phase 必須滿足）

- **計算整合**：各引擎結果必須融入 `IntegratedFortuneData`，影響 `overallScore`、`investmentScore`、`recommendation`
- **UI 整合**：各引擎結果必須在 Dashboard 有對應展示區塊，根據啟用狀態動態顯示/隱藏
- **歷史整合**：各引擎結果必須寫入 FortuneLogViewer 的 IndexedDB 歷史記錄
- **設定整合**：各引擎的開關和權重必須在 Profile 頁面可調整，並持久化到 localStorage
- **向後兼容**：現有功能（農民曆、五行雷達圖、交易時段、股價走勢）不受影響
